1. **`ifconfig`**:
   - **Description**: Displays or configures network interface parameters for a network using the Internet Protocol. It shows information about network interfaces such as IP addresses, MAC addresses, and network statistics.
   - ![[CCNA/cn-archive/docs/images/Pasted image 20240909002730.png]]

2. **`ip`**:
   - **Description**: A versatile tool for network management that replaces older tools like `ifconfig`. It is used to show and manipulate routing, devices, policy routing, and tunnels.
   - ![[CCNA/cn-archive/docs/images/Pasted image 20240909002823.png]]

3. **`traceroute`**:
   - **Description**: Shows the route packets take from one host to another. It lists each hop along the path and helps in diagnosing network routing issues.
   - ![[CCNA/cn-archive/docs/images/Pasted image 20240909003333.png]]

4. **`tracepath`**:
   - **Description**: Similar to `traceroute`, it traces the network path to a target host and displays the route packets take, including any network issues or packet loss.
   - ![[CCNA/cn-archive/docs/images/Pasted image 20240909003540.png]]

5. **`ping`**:
   - **Description**: Sends ICMP Echo Request messages to a host to check its reachability and measure round-trip time. It's useful for checking if a network device is reachable and how long it takes to communicate.
   - ![[CCNA/cn-archive/docs/images/Pasted image 20240909003617.png]]

6. **`netstat`**:
   - **Description**: Displays network connections, routing tables, interface statistics, masquerade connections, and multicast memberships. It's useful for network troubleshooting and monitoring.
   - ![[CCNA/cn-archive/docs/images/Pasted image 20240909003641.png]]

7. **`nslookup`**:
   - **Description**: Queries DNS to obtain domain name or IP address mapping. It helps in diagnosing DNS-related issues by providing details about DNS records.
   - ![[CCNA/cn-archive/docs/images/Pasted image 20240909003658.png]]

8. **`dig`**:
   - **Description**: A flexible tool for querying DNS nameservers to retrieve domain name information. It provides detailed information about DNS records and is often used for troubleshooting DNS issues.
   - ![[CCNA/cn-archive/docs/images/Pasted image 20240909003716.png]]

9. **`route`**:
   - **Description**: Displays or modifies the IP routing table. It is used to view or configure routes for network traffic and can help in diagnosing routing issues.
   - ![[CCNA/cn-archive/docs/images/Pasted image 20240909003731.png]]

10. **`host`**:
    - **Description**: Performs DNS lookups to resolve domain names to IP addresses and vice versa. It's a simpler alternative to `dig` for querying DNS information.
    - ![[CCNA/cn-archive/docs/images/Pasted image 20240909003756.png]]

11. **`arp`**:
    - **Description**: Displays and modifies the ARP (Address Resolution Protocol) cache. This cache maps IP addresses to MAC addresses on a local network.
    - ![[CCNA/cn-archive/docs/images/Pasted image 20240909003833.png]]

12. **`iwconfig`**:
    - **Description**: Configures wireless network interfaces. It provides information about the wireless connection, such as signal strength and access points.
	    - ![[CCNA/cn-archive/docs/images/Pasted image 20240909003903.png]]

14. **`wget`**:
    - **Description**: A command-line utility for downloading files from the web. It supports HTTP, HTTPS, and FTP protocols and can download multiple files and directories recursively.
    - ![[CCNA/cn-archive/docs/images/Pasted image 20240909004150.png]]

15. **`telnet`**:
    - **Description**: Provides a command-line interface for communicating with a remote host using the Telnet protocol. It is used for testing and debugging network services.
    - ![[CCNA/cn-archive/docs/images/Pasted image 20240909005511.png]]

16. **`whois`**:
    - **Description**: Queries domain registration information from a WHOIS database. It provides details about domain ownership, registration, and contact information.
    - ![[CCNA/cn-archive/docs/images/Pasted image 20240909005829.png]]

17. **`ifplugstatus`**:
    - **Description**: Displays the status of network interfaces and their connection state. It is useful for checking if an Ethernet cable is plugged in or not.
    - ![[CCNA/cn-archive/docs/images/Pasted image 20240909005906.png]]

18. **`nload`**:
    - **Description**: Monitors network traffic and bandwidth usage in real-time. It provides a graphical representation of incoming and outgoing traffic on network interfaces.
    - ![[CCNA/cn-archive/docs/images/Pasted image 20240909005945.png]]
